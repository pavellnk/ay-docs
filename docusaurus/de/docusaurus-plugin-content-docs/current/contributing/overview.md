# Zu Ayunis Core beitragen

Vielen Dank für Ihr Interesse, zu Ayunis Core beizutragen! Dieser Leitfaden hilft Ihnen beim Einstieg.

## Möglichkeiten zum Beitragen

Es gibt viele Möglichkeiten beizutragen:

- **Fehler melden** - Öffnen Sie ein Issue, das das Problem beschreibt
- **Funktionen vorschlagen** - Teilen Sie Ihre Ideen für Verbesserungen
- **Dokumentation verbessern** - Beheben Sie Tippfehler, fügen Sie Beispiele hinzu oder schreiben Sie neue Anleitungen
- **Code einreichen** - Beheben Sie Fehler oder implementieren Sie neue Funktionen

## Entwicklungsumgebung einrichten

### Voraussetzungen

- Node.js 18+
- Git
- Ein Code-Editor (wir empfehlen VS Code)

### Lokale Entwicklung

1. Forken Sie das Repository
2. Klonen Sie Ihren Fork:

```bash
git clone https://github.com/IHR_BENUTZERNAME/ayunis-core.git
cd ayunis-core
```

3. Installieren Sie die Abhängigkeiten:

```bash
npm install
```

4. Erstellen Sie einen Branch für Ihre Änderungen:

```bash
git checkout -b feature/ihr-feature-name
```

## Änderungen einreichen

### Pull-Request-Prozess

1. Stellen Sie sicher, dass Ihr Code alle Tests besteht
2. Aktualisieren Sie die Dokumentation nach Bedarf
3. Schreiben Sie eine klare PR-Beschreibung
4. Fordern Sie eine Überprüfung von den Maintainern an

### Commit-Nachrichten

Wir folgen [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: neue Funktion hinzufügen
fix: Fehler beheben
docs: Dokumentation aktualisieren
chore: Wartungsaufgaben
```

## Verhaltenskodex

Bitte seien Sie respektvoll und inklusiv in allen Interaktionen. Wir sind bestrebt, eine einladende Umgebung für alle zu schaffen.

## Fragen?

Wenn Sie Fragen haben, können Sie gerne:

- Eine GitHub Discussion öffnen
- In unseren Community-Kanälen fragen

Wir schätzen Ihre Beiträge!
